# 中→英 智能翻译 PWA

基于 DeepSeek API 的中译英翻译工具，核心特色是**低 AI 检测率**——翻译结果自然流畅，不易被 AI 检测工具识别。

## 核心技术：智能流水线

```
输入中文
  → [第1步] 拟人化翻译（反AI检测策略 + 深度提示词工程）
  → [第2步] AI痕迹检测（七维度评估体系）
  → [第3步] 超标自动改写（外科手术式去AI）
  → [第4步] 二次检测 → (仍超标? 深度改写)
  → 输出自然英文
```

- **改写阈值**：AI评分 > 25 分自动触发改写
- **深度改写**：改写后若评分 > 60 分，自动进行第二轮更高强度的深度改写
- **改写温度**：翻译 0.75 / 初次改写 0.85 / 深度改写 0.95（逐步增加多样性）

## 快速开始

### 1. 前提条件

- [DeepSeek API Key](https://platform.deepseek.com/)（免费注册即可获取）

### 2. 部署

**方案 A：GitHub Pages（推荐）**
1. Fork 本仓库
2. Settings → Pages → 选 `main` 分支 + `/docs` 目录 → Save
3. 等待几分钟即可通过 `https://<你的用户名>.github.io/translate-miniapp/` 访问

**方案 B：本地运行**
1. 在项目目录下启动任意静态文件服务器，例如：
   ```
   npx serve docs
   ```
2. 浏览器打开 `http://localhost:3000`

### 3. 使用

1. 打开网页，首次访问会弹出设置面板
2. 填入 DeepSeek API Key（存储在浏览器 localStorage 中）
3. 输入中文 → 点击翻译，三步流水线自动执行
4. 翻译历史自动保存在本地

## 技术方案

- **单文件 PWA**：HTML + CSS + JS 全部内联，零依赖
- **直连 DeepSeek API**：浏览器端直连，无需后端/云函数
- **localStorage 持久化**：翻译历史本地存储
- **PWA 可安装**：手机端添加到桌面，体验接近原生 App
- **响应式设计**：一套代码适配手机/平板/桌面

## 设计说明

### 翻译策略
- 角色扮演（15年翻译专家）+ 意译优先 + 句式多变
- **反AI检测策略**：强制句式破坏（长短交替）、禁用 `This + verb` 机械链、禁用 `is + 程度副词 + adj` 模式、段落变形、禁用标题大小写和FAQ模板
- 翻译温度 0.75，兼顾质量与多样性

### 检测维度（七维度评估体系）

| 维度 | 权重 | 检测内容 |
|------|------|----------|
| 句式均匀度 | 20% | 相邻句长差异、开头方式多样性 |
| 过渡词与连接词 | 15% | AI高频连接词 + 段落开头模式 |
| 翻译腔特征 | 15% | of结构、被动语态密度、程度副词滥用 |
| 模板化短语 | 15% | `This + verb` 链、AI语料库高频搭配 |
| 结构模板化 | 15% | FAQ模式、编号列表、段落均匀度、标题大小写 |
| 自然度 | 10% | 习语、缩写、口语化表达 |
| 信息密度比 | 10% | 实义信息 vs 填充性文字 |

### 改写策略
- **外科手术式**：对AI特征句做结构性改造，而非微调
- **5层优先级**：句子手术 → 词汇替换 → 结构破坏 → 添加人类特征 → 信息密度提升
- 内置AI过渡词→自然表达替换表（如 Moreover→Plus / Therefore→So）
- 仅修改30-50%文本，保留原文骨架

### 流水线逻辑
- 翻译 → 检测(一次) → (score>25?) 改写 → 检测(二次) → (score>60?) 深度改写 → 完成
- 改写温度逐步提升，深度改写使用最高温度以打破顽固AI模式
